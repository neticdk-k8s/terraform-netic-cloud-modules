# Container Registry — OVHcloud

Provisions an OVHcloud Managed Container Registry with optional user accounts and IP access restrictions.

## Resources created

| Resource | Condition | Description |
|----------|-----------|-------------|
| `ovh_cloud_project_containerregistry` | `deploy = true` | The container registry |
| `ovh_cloud_project_containerregistry_ip_restrictions_management` | `deploy = true` + `ip_restrictions` non-empty | Allowlist for registry access |
| `ovh_cloud_project_containerregistry_user` | `deploy = true` + `registry_users` non-empty | Registry user accounts |

## Usage

### Basic registry

```hcl
module "registry" {
  source = "./modules/container_registry/ovh"

  ovh_project_id = var.ovh_project_id

  container_registry = {
    deploy = true
    name   = "my-registry"
    region = "GRA"
  }
}
```

### Registry with users and IP restrictions

```hcl
module "registry" {
  source = "./modules/container_registry/ovh"

  ovh_project_id = var.ovh_project_id

  container_registry = {
    deploy = true
    name   = "my-registry"
    region = "GRA"
  }

  registry_users = [
    { login = "ci-user",  email = "ci@example.com" },
    { login = "dev-user", email = "dev@example.com" }
  ]

  ip_restrictions = [
    { ip_block = "203.0.113.0/24", description = "Office network" },
    { ip_block = "198.51.100.5/32", description = "CI runner" }
  ]
}

output "registry_url" {
  value = module.registry.registry_url
}

output "user_passwords" {
  value     = module.registry.user_passwords
  sensitive = true
}
```

## Inputs

| Name | Type | Description |
|------|------|-------------|
| `ovh_project_id` | `string` | OVH Public Cloud project ID |
| `container_registry` | `object` | Registry configuration (see below) |
| `registry_users` | `list(object)` | User accounts to create (default: `[]`) |
| `ip_restrictions` | `list(object)` | IP allowlist (default: `[]` — no restrictions) |

### `container_registry`

| Field | Type | Description |
|-------|------|-------------|
| `deploy` | `bool` | Set to `false` to skip resource creation (useful for conditional environments) |
| `name` | `string` | Registry name |
| `region` | `string` | OVH region (e.g. `"GRA"`) |

### `registry_users` list entry

| Field | Type | Description |
|-------|------|-------------|
| `login` | `string` | Username |
| `email` | `string` | User email address |

### `ip_restrictions` list entry

| Field | Type | Description |
|-------|------|-------------|
| `ip_block` | `string` | CIDR block to allow |
| `description` | `string` | Human-readable label for the rule |

## Outputs

| Name | Description |
|------|-------------|
| `registry_url` | Public URL of the container registry (null if `deploy = false`) |
| `user_passwords` | Map of username → generated password *(sensitive)* |

## Provider

```hcl
provider "ovh" {
  endpoint           = "ovh-eu"
  application_key    = var.ovh_application_key
  application_secret = var.ovh_application_secret
  consumer_key       = var.ovh_consumer_key
}
```

## Notes

- **User passwords** are generated by OVH and returned in the `user_passwords` output. Store them in a secrets manager — they cannot be retrieved again without recreating the user.
- **IAM vs. local users** — OVH supports either IAM-based access or local registry users, but not both simultaneously. This module uses local users. Comment out `registry_users` and configure IAM separately if you prefer the IAM approach.
- **`deploy = false`** — All resources are skipped and outputs return `null`. Useful when the same Terraform configuration targets multiple environments where the registry is not always needed.
